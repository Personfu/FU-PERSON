# ============================================================================
#  FLLC - FU PERSON | EMV Contactless Card Reference
# ============================================================================
#  Reference guide for reading and analyzing contactless payment cards.
#  Flipper Zero can READ EMV data but CANNOT clone/replay payment cards.
# ============================================================================

# ============================================================================
#  EMV DATA FIELDS (What Flipper Captures)
# ============================================================================
#
#  TAG      | NAME                    | NOTES
#  ---------|-------------------------|----------------------------------------
#  5A       | PAN (Card Number)       | Usually partial (last 4-8 digits)
#  5F24     | Expiry Date             | YYMMDD format
#  5F20     | Cardholder Name         | May be present or empty
#  9F26     | App Cryptogram          | Dynamic, single-use
#  9F27     | Cryptogram Info Data    | Indicates type (ARQC/TC/AAC)
#  9F10     | Issuer App Data         | Issuer-specific data
#  9F36     | App Transaction Counter | Increments per transaction
#  9F6C     | Card Transaction Qual   | Indicates contactless capabilities
#  57       | Track 2 Equivalent      | PAN + expiry for magstripe fallback
#  82       | AIP                     | Application Interchange Profile
#  94       | AFL                     | Application File Locator
#  9F37     | Unpredictable Number    | Random challenge
#

# ============================================================================
#  WHAT YOU CAN DO WITH EMV DATA
# ============================================================================
#
#  1. OSINT/RECONNAISSANCE:
#     - Identify cardholder name
#     - Determine card network (Visa/MC/Amex)
#     - Determine issuing bank (from BIN/IIN)
#     - Identify card type (credit/debit/prepaid)
#     - Track 2 data for social engineering
#
#  2. BIN LOOKUP:
#     First 6-8 digits of PAN = BIN (Bank Identification Number)
#     Use binlist.net or similar to identify:
#       - Issuing bank
#       - Card type
#       - Card level (Gold, Platinum, Black)
#       - Country of issue
#
#  3. WHAT YOU CANNOT DO:
#     - Clone the card (dynamic cryptograms prevent replay)
#     - Make purchases (CVV not on chip, cryptogram is one-time)
#     - Extract PIN
#     - Bypass EMV authentication
#
# ============================================================================
#  FLIPPER WORKFLOW FOR EMV
# ============================================================================
#
#  1. Flipper -> NFC -> Read
#  2. Hold near target's wallet/pocket (< 4cm, 1.5in)
#  3. Flipper identifies card type and reads available data
#  4. Data saved to SD card
#  5. Review in: SD/nfc/<saved_file>.nfc
#
#  TIPS:
#  - Metal wallets / RFID blockers will prevent reading
#  - Through jeans pocket: ~2cm range
#  - Through thin wallet: ~3cm range
#  - Bus/train: Stand close during crowded times
#  - Cafe: While "looking at phone" near their wallet on table
#
