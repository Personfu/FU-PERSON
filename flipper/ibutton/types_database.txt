# ============================================================================
#  MOST COMMON: DS1990A (Family 01h) â€” ACCESS CONTROL
# ============================================================================
#
#  The DS1990A is the #1 iButton for physical access control.
#  It contains ONLY a 48-bit unique serial number (no memory, no crypto).
#
#  UID FORMAT: [Family Code(8)][Serial Number(48)][CRC(8)] = 8 bytes total
#  Example: 01:A2:B3:C4:D5:E6:00:CRC
#
#  SECURITY: ZERO. The serial number is broadcast in plaintext.
#  Read it once -> emulate forever.
#
#  Flipper Zero workflow:
#  1. iButton -> Read
#  2. Touch Flipper's iButton pad to the target key
#  3. Flipper reads the full 8-byte UID
#  4. Save -> Emulate (Flipper acts as the key)
#  5. Or Write to blank RW1990/TM1990 clone button

#    Many intercoms accept ANY DS1990A key that's been registered.

#    Some buildings restrict elevator access using iButtons.
#    Clone a resident's key to access restricted floors.

#    Read the attendant's master key for unlimited access.

#  SCENARIO 4: LAUNDRY ROOM

#  SCENARIO 5: GYM / POOL ACCESS
#    Fitness facilities often use basic iButton readers.
#    DS1990A with no encryption = instant clone.

# ============================================================================
#  iBUTTON READ TECHNIQUES

#  - Apply firm pressure for good electrical contact
#  - If read fails, rotate the iButton slightly and retry
#  - Clean both contacts with alcohol if dirty
#  - Read distance: ZERO (must make physical contact)
